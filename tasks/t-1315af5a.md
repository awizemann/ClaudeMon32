## Description

SYMPTOM: panel flashes then WAITING FOR HOST periodically (~15-30 min, irregular) = device REBOOT. Started AFTER WiFi enabled; rock-solid for hours on serial before.

CONFIRMED (serial 2026-07-13): Guru panic, rst:0xc, backtrace = multi_heap_free (heap poisoning) <- free <- esf_buf_recycle <- ppTask (WiFi stack). NOT brownout.

ROOT CAUSE (high confidence, 3-agent + config analysis):
- Live instrumentation shows internal heap HEALTHY and FLAT (~170KB free, 147KB largest, ok=1) — NOT a leak/starvation. So shrinking LV_MEM_SIZE is NOT the fix.
- No C-style OOB write in our code (all snprintf stack-bounded; LVGL uses its own internal pool; no cross-core races — everything runs on core 1).
- The prebuilt arduino-esp32 2.0.17 qio_opi sdkconfig (THIS build) ships CONFIG_ESP32S3_DATA_CACHE_LINE_SIZE=32 (WRONG for octal PSRAM's 64-byte bursts) AND CONFIG_SPIRAM_TRY_ALLOCATE_WIFI_LWIP=1 (WiFi/LWIP dynamic buffers go to PSRAM). WiFi buffers in PSRAM get small cached CPU writes corrupted by the 32-byte line on 64-byte octal PSRAM -> esf_buf_recycle free trips the poison assert -> panic. Explains: only with WiFi (framebuffers are large aligned DMA writes and survive; WiFi buffers don't), internal heap healthy (corruption is in PSRAM), display fine on serial.

FIX OPTIONS (the bad values are baked into precompiled arduino libs; can't be changed by build_flags):
1. Recompile arduino-esp32 libs with corrected sdkconfig (SPIRAM_TRY_ALLOCATE_WIFI_LWIP=n + DATA_CACHE_LINE_SIZE=64) while keeping the arduino 2.0.x API — via tasmota/platform-espressif32 or IDF+arduino-as-component. Fixes root cause, our code unchanged. RECOMMENDED. Medium effort/risk (build-system change).
2. Migrate to pioarduino (arduino 3.x / IDF 5.x): whole bug class fixed upstream + correct octal cache handling, but a major API jump (LovyanGFX/LVGL/WiFi) with high regression risk.
3. Interim: gate WiFi off by default (device stable on serial, WiFi opt-in) until 1/2 lands.
Secondary cheap cleanup regardless: mDNS begin-once/never-end (agent flagged MDNS.end()/begin() churn on reassociation as a use-after-free risk); WiFi.setTxPower lower.
Instrumentation (heap logging + reset_reason + integrity check) is in main.cpp on the branch (uncommitted) — keep for the fix session.
