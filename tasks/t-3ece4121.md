## Description

THE risky core. Anthropic refresh tokens rotate on every refresh and are shared with Claude Code, so refreshing a copied token invalidates the desktop session. Design direction: the device performs its OWN OAuth login (device/PKCE flow) to obtain a separate grant, so its rotation never collides with the desktop. Needs the OAuth flow implemented on the ESP32 + refresh scheduling + NVS token storage. Do a dedicated design pass before building. Depends on P4.0 (NVS) and ideally P4.2 (a UI to trigger login).
