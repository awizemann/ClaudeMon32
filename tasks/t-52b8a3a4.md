## Description

Net.cpp's TCP command server holds a single client and only accepts a new one when the current s_client reports !connected(). When the host daemon is killed/restarted, the device can keep the stale half-open client for a bit, so the new daemon's WiFi connect times out (~3s) and AutoLink falls back to serial until the next reconnect re-prefers WiFi. Self-heals, and a clean device reboot has no stale client (WiFi is picked cleanly) — so low priority. Improvement: on a new incoming connection, prefer it (drop/replace the old client), or ping-timeout the existing client sooner. esp32/crowpanel/src/Net.cpp (net_loop client handling).
